Interview Exercise

By Carrow Morris-Wiltshire

Key challenge: production line downtime


Basic Assumptions

  • Atleast one factory has data 'fit for purpose'
  • Sensors are IoT sensors
  • Maintenance teams will adopt and trust the new system

Proposed Solution

3 Phases

Phase 1: Data Integration and Standardisation

  • Standardise sensor measurement frequencies
  • Create a structured failure reporting template
  • Implement automated data validation
  • Centralised data lake
  • Real-time sensor data pipeline

Ensure standardised template

Range checks, missing data detection, sampling rate verification, cross-validation between sensor types - make sure the sensors are working.

Ambient temperature and humidity, load variations, machine startup/shutdown cycles, product type being manufactured

Phase 2: MVP Development

Target: Single factory pilot program

  1. Feature Engineering
    • Rolling statistics on sensor data
    • Text mining on historical work orders
    • Domain specific indicators (e.g. MTBF)
  2. Initial Model Development
    • Simple anomaly detection (isolation forest)
    • Basic time-series forecasting (ARIMA)
    • Performance baseline establishment

Moving averages of sensor readings, rolling standard deviation, rate of change calculation

Establishing a baseline for "normal" failure frequency, identifying if failures are becoming more frequent (decreasing MTBF)

Phase 3: Advanced Analytics

  1. Hybrid Model Architecture
    • Deep learning for sensor patterns (LSTM)
    • Physics informed contraints
    • Multi-sensor fusion
  2. Evaluation Framework
    • False positive / negative analysis
    • Repair lead time optimisation
    • Cost / benefit analysis

Capturing long-term temporal dependencies

Heat transfer equations governing motor temperature evolution, mechanical equations for bearing dynamics and vibrations, conservation of energy relating input power to output work

LSTM learns physically realistic failure progressions, reducing false positives, physics loss term penalises violations during training

Correlations between different sensor types, cascade failure prediction

Implementation Strategy

  1. Start with the highest-volume factory
  2. Continuous validation with maintenance teams
  3. Iterative improvement based on feedback
  4. Gradual rollout to other factories
